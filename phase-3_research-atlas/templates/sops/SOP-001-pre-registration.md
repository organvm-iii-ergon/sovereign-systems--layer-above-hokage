# SOP 001: Study Pre-Registration

## Status: ACTIVE | Version: 1.0

**Purpose**: Lock hypotheses and analysis plans before data collection.

---

## Procedure

### 1. PREPARE (Day 0)

```
□ Finalize study ID: SOVEREIGN-[YEAR]-[###]-[PILLAR]
□ Document hypothesis (H1, H0)
□ Calculate sample size
□ Define metrics + sources
□ Choose statistical method
□ Prepare analysis code (locked)
```

### 2. REGISTER (Day 1)

```
□ Upload to OSF/GitHub
□ Generate timestamped URL
□ Record in experiment tracker
□ Notify team of pre-registration
```

### 3. EXECUTE (Days 2-N)

```
□ Collect data per protocol
□ Document any deviations
□ DO NOT analyze until locked
```

### 4. ANALYZE (Day N+1)

```
□ Run locked analysis code
□ Run sensitivity analyses
□ Record all results (positive + negative)
```

### 5. REPORT (Day N+7)

```
□ Write findings
□ Submit for internal review
□ Update evidence base
□ Close study in tracker
```

---

## Exceptions

| Situation | Action |
|---|---|
| Protocol deviation | Document immediately, decide if valid |
| Analysis error | New pre-registration required |
| Equipment failure | Resume after fix, document |

---

## Quality Gate

| Check | Requirement |
|---|---|
| Hypothesis | Falsifiable, explicit |
| Sample size | Power ≥ 0.80 |
| Metrics | Operationalized with sources |
| Analysis | Code runs without error |
